Anatomy of an HTML element

HTML is a markup language that defines the structure of your content. HTML consists of a series of elements, which you use to enclose, or wrap, different parts of the content to make it appear a certain way, or act a certain way. The enclosing tags can make a word or image hyperlink to somewhere else, can italicize words, can make the font bigger or smaller, and so on. HTML consists of tags like <h1> (for heading) and <p> (for paragraph).

There are four main parts that you see above:

  • The opening tag, wrapped in opening and closing angle brackets.
  • The closing tag, also wrapped in opening and closing brackets but it also includes a forward slash BEFORE the element name that states this is where the element ends.
  • The content, which in this case is just text.
  • The element, which is the entire thing together as a whole.

Elements can also have attributes such as the one below:

Attributes should always have a space between it and the element name, the attribute name followed by an equal sign, and the attribute value wrapped in quotation marks. You will see different developers use single and/or double quotes. This is a matter of style. You can feel free to choose whichever you prefer, but it is best practice to stick with one style in an entire project.


Elements can also be put inside other elements, which is called nesting. An example would be using <b> to make text bold.

Starting Your HTML Document

  • <!DOCTYPE html> is a required preamble. It is needed to make sure your document behaves correctly.
  • <html> This element wraps all the content on the entire page and is sometimes known as the root element.
  • <head> This element acts as a container for the things you need to include on the HTML page that isn't shown to your viewers.
  • <meta charset="utf-8"> This element sets the character set of your document to UTF-8, which includes most characters from the vast majority of written languages.
  • <meta name="viewport" content="width=device-width"> This viewport element ensures the page render at the width of viewport, preventing mobile browsers from rendering pages wider than the viewport and then shrinking them down.
  • <title> This sets the title of your page that appears in the browser tab and is used to describe the page when it's bookmarked.
  • <body> This contains all the content you want to show to your viewers when they visit your page.

Text Fundamentals

Most structured text consists of headings and paragraphs, whether you are reading a story, a newspaper, a college textbook, a magazine, etc. Structured content makes the reading experience easier and more enjoyable.

Headings

There are six heading elements from <h1> to <h6>. <h1> is used for the main heading and <h2> and the others respectively are used as subheadings and sub-subheadings.


Because users tend to scan quickly through text to find relevant content, oftentimes reading headings first, you want to display your information with useful titles and subtitles.

Lists

There are unordered <ul> and ordered <ol> lists. Within each, you have the list items, designated by <li>

Hyperlinks

A basic hyperlink is created with the <a> element using the href attribute, also known as a Hypertext Reference, or target, that contains the destination you want the link to go to. You may also want to add a title attribute in order to detail the information about the link, such as what the page contains and where it's leading the viewer to.

Web development has a number of pros: High earning potential, opportunity for growth, and flexibility are usually top of mind for most who get into the industry. Some of the challenges for people who pursue web development is the competitive nature of the job market. Once you know what web development is, you may wonder if web development is worth learning. Well, yes, it can be worth learning if you enjoy problem-solving and technology, and you want to learn a modern skill that can help you in the job market.

In web development, you will learn front-end frameworks, back-end frameworks, programming languages as well as design patterns, authentication, APIs, deployment, and more. There are no steps that tell you you know everything about web development. The technologies that power the bespoke web software domain represent the future of web development. PWA, SPA, serverless architecture, and Motion UI will be particularly prominent web trends in 2022 and beyond.

Unsurprisingly, web development is still a lucrative field in 2022. Companies continue to place great emphasis on hiring quality developers who understand the modern web tech stack. Website development is a way to make people aware of the services and/or products you are offering, understand why your products are relevant and even necessary for them to buy or use, and see which of your company's qualities set it apart from competitors.
